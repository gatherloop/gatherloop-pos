# Who It's For

This site serves three different readers. Pick the one that's you.

## Coffee shop staff & owners

**You want to know:** *Can this run my shop? What does each screen do for me?*

Every feature page in the sidebar is written in plain language and organized by the job you're doing — selling, restocking, or tracking money — not by how the code is structured. Start with [Sales & Checkout](/sales/transactions) for the counter experience, then [Inventory](/inventory/stock-checks) and [Finance](/finance/dashboard-statistics) for the back-office side.

## Investors

**You want to know:** *Is this a real, complete product? How broad is the surface area? Is it thoughtfully built?*

The sidebar itself is the pitch: one glance shows the full feature map across sales, catalog, inventory, finance, and operations — all shipped, not planned. Start with [The Big Picture](./the-big-picture) for the end-to-end loop the product runs on, then check the [Roadmap](/roadmap) for what's shipped, in progress, and planned next.

## Recruiters & hiring engineers

**You want to know:** *Can this person design and ship a non-trivial system?*

The [Under the Hood](/under-the-hood/architecture) section is written for you: architecture diagrams, tech stack rationale, a Clean Architecture write-up for both the backend and frontend, the cross-platform (web + mobile) approach, and the testing strategy — each page links into the real source at [github.com/gatherloop/gatherloop-pos](https://github.com/gatherloop/gatherloop-pos).
